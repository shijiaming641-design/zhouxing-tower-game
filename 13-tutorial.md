# 教学关卡设计

## 1. 第1层教学关概述

### 1.1 设计目标

| 目标 | 说明 |
|------|------|
| 核心教学 | 让玩家理解猜拳基本机制 |
| 战斗流程 | 熟悉回合制战斗操作 |
| UI认知 | 认识战斗界面各元素 |
| 心理预期 | 建立"策略vs运气"的认知 |

### 1.2 关卡配置

| 项目 | 内容 |
|------|------|
| 层数 | 第1层 |
| 敌人 | 蒸汽劳工×1 |
| 难度 | 简单（AI: 完全随机） |
| 强制战斗 | 是（不可跳过） |

## 3. 教学关敌人配置

### 3.1 蒸汽劳工（教学敌人）

| 属性 | 数值 |
|------|------|
| 生命值 | 20（降低） |
| 攻击力 | 5（降低） |
| 防御力 | 0 |
| AI | 完全随机 |
| 特殊效果 | 无 |

**设计意图**：
- 极低数值，让玩家轻松获胜
- 完全随机出拳，避免玩家感觉"被设计"
- 胜利后建立信心

## 4. 教学提示系统

### 4.1 提示触发条件

| 场景 | 提示内容 |
|------|----------|
| 首次进入战斗 | "点击石头/剪刀/布进行出拳" |
| 3秒未操作 | "选择你的出拳" |
| 连续2次平局 | "平局时伤害较低，尝试克制敌人" |
| 连续2次失败 | "观察敌人的出拳倾向，预测下一拳" |
| 使用技能 | "技能需要消耗怒气，谨慎使用" |

### 4.2 提示样式

```
[气泡提示]
┌──────┐
│ 💡   │
│ 内容  │
└──────┘
     ↓ 指向相关UI区域
```

## 8. 教学难度控制

### 8.1 可跳过教学

| 教学项 | 可跳过 | 跳过条件 |
|--------|--------|----------|
| 三元素介绍 | 是 | 曾有RPS游戏经验 |
| 战斗流程 | 是 | 曾在教学中获胜 |
| 商店介绍 | 是 | 曾有卡牌游戏经验 |

### 8.2 难度设置

| 难度 | 敌人AI | 敌人数值 | 奖励 |
|------|--------|----------|------|
| 简单(教程) | 纯随机 | 降低50% | 固定 |
| 普通 | 倾向+随机 | 正常 | 随机 |
| 困难 | 学习型 | 提升20% | 丰富 |
